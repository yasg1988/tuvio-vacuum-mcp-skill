# Получение авторизации пылесоса

Перед установкой MCP-сервера нужен уже работающий локальный хаб или другой локальный HTTP API, который умеет отправлять команды Tuya-пылесосу. Сам MCP-сервер из этого репозитория не хранит Tuya-секреты: он вызывает локальный хаб.

## Что нужно получить

Для Tuya LAN обычно нужны:

- `device_id` - ID устройства;
- `local_key` - локальный ключ устройства;
- `product_id` - ID продукта, необязателен для MCP, но полезен для диагностики;
- IP пылесоса в локальной сети.

Эти значения нельзя коммитить в GitHub. Храните их только локально: в состоянии хаба, `.env`, менеджере секретов или другом приватном конфиге.

## Вариант 1. Tuya IoT Platform

1. Установите приложение, через которое виден пылесос: Smart Life, Tuya Smart, Tuya Home или OEM-приложение производителя, если оно совместимо.
2. Откройте Tuya IoT Platform: `https://platform.tuya.com/cloud/`.
3. Создайте Cloud Project типа Smart Home.
4. Выберите тот дата-центр, где реально находится аккаунт приложения. Для аккаунтов Россия/Казахстан часто подходит Central Europe Data Center (`EU`), но проверять нужно по факту привязки.
5. В проекте откройте привязку аккаунта приложения и покажите QR-код.
6. Отсканируйте QR-код приложением Smart Life/Tuya/Tuya Home.
7. Убедитесь, что в проекте появилось устройство пылесоса.
8. Получите `device_id` и `local_key` через официальный API/Device Management или через утилиту `tinytuya wizard`.

Если устройство видно в приложении, но не появляется в Tuya IoT Project, проверьте регион проекта и регион аккаунта приложения.

## Вариант 2. Через уже настроенный локальный хаб

Если у вас уже есть локальный хаб, который хранит `device_id` и `local_key`, MCP-серверу эти секреты не нужны. Ему нужен только URL хаба:

```text
VACUUM_HUB_URL=http://127.0.0.1:8790
```

Хаб должен поддерживать:

- `GET /api/devices` - вернуть состояние, включая `tuya_live.named`;
- `POST /api/tuya/command` - принять JSON `{"command":"start"}` и выполнить команду.

## Проверка перед установкой MCP

Проверьте локально:

```powershell
Invoke-WebRequest -UseBasicParsing http://127.0.0.1:8790/api/devices
```

Проверьте команду без опасных действий, например `find`:

```powershell
Invoke-WebRequest -UseBasicParsing http://127.0.0.1:8790/api/tuya/command `
  -Method POST `
  -ContentType "application/json" `
  -Body '{"command":"find"}'
```

Если эти запросы работают, MCP-сервер можно подключать.

## Важные ограничения

- Не публикуйте `local_key`, пароль роутера, Tuya Client Secret, QR-токены и cookies.
- Не кладите в репозиторий `hub_state.json`, дампы ответов Tuya Cloud, QR-коды и временные файлы авторизации.
- Если пылесос добавлен только в OEM-приложение Tuvio и не виден в Smart Life/Tuya Home, получение `local_key` может потребовать привязки через Tuya IoT Platform с правильным регионом.
